Questionário

 Questão 1: Explique qual é a função de um servidor web em um ambiente de hospedagem e cite pelo menos um exemplo.

R: Lidar com requisições HTTP/HTTPS e servem conteúdo estático (HTML, CSS, JS, imagens). Exemplo: Apache HTTP Server.
Questão 2: Qual a principal característica do NGINX e por que ele é muito utilizado em projetos de alto desempenho?

R: Leve e eficiente. Porque é ótimo para alto desempenho e proxy reverso, conseguindo lidar com muitas conexões sem apresentar problemas.
Questão 3: Diferencie os protocolos HTTP e HTTPS. Por que o HTTPS é recomendado para aplicações modernas?

R: O HTTPS tem criptografia SSL/TLS. Porque oferece maior segurança dos dados fornecidos.
Questão 4: Qual é a função das portas em uma rede de computadores e qual a importância da porta 443?

R: São canais de comunicação entre esses computadores. A porta 443 fornece uma maior segurança dos dados, já que utiliza protocolo HTTPS que os criptografa.
Questão 5: Defina o conceito de endereço IP e explique sua importância em redes de computadores.

R: É um número identificador de um dispositivo na rede. São importantes porque permitem que os dados sejam enviados corretamente.
Questão 6: Explique o que é hospedagem VPS e em quais situações ela é mais indicada.

R: É a hospedagem em um servidor virtual, com mais controle e desempenho. Ideal para sites que superaram os limites da hospedagem compartilhada, mas ainda não exigem todo o poder de um servidor físico exclusivo.
Questão 7: Descreva as vantagens do servidor web Caddy em relação a outros servidores tradicionais.

R: Servidor moderno com HTTPS automático e configuração simples.
Questão 8: Qual servidor de aplicação é mais indicado para aplicações desenvolvidas na plataforma .NET (C#)? Explique o motivo.

R: ASP.NET Core. Suas ferramentas suportam a linguagem C#.
Questão 9: Descreva o funcionamento do modelo Cliente-Servidor no contexto de sistemas web.

R: Modelo onde o cliente (navegador ou app) envia requisições a um servidor, que responde com os dados/processamento solicitado.
Questão 10: Explique o que é o Apache Tomcat e para qual tipo de aplicação ele é mais indicado.

R: Servidor de aplicação que implementa servlets e JSP. Muito usado com aplicações Java.
Questão 11: Diferencie, com suas palavras, um servidor web de um servidor de aplicação, destacando suas funções principais.

R: A função de um servidor web é lidar com a segurança e entregar conteúdo que não se altera, como imagens. Já o servidor de aplicação processa e gera os componentes dinâmicos do site, os quais dependem de lógica de programação para permitir a interatividade do usuário com a página.
Questão 12: Defina o conceito de domínio na internet e sua relação com os endereços IP.

R: Nome que representa um IP. Ex: www.fluminense.com. São representações daquele endereço IP.
Questão 13: Quais as principais vantagens da hospedagem em nuvem em comparação com a hospedagem local?

R: A hospedagem em nuvem é distribuída, escalável e com alta disponibilidade e não necessita de um espaço físico.
Questão 14: Cite dois exemplos de servidores de aplicação e as linguagens/plataformas que eles suportam.

R: Django (com Gunicorn/WSGI), que suporta Python; e Node.js, que suporta Javascript.
Questão 15: Por que é essencial a utilização de HTTPS em sites e sistemas atualmente? Quais riscos o HTTP apresenta?

R: O HTTPS tem criptografia SSL/TLS, que confere segurança e confidencialidade à troca de dados entre o usuário e o servidor. Como HTTP não apresenta criptografia, a segurança dos dados fica comprometida.
Questão 16: Dê um exemplo de situação onde o uso de hospedagem VPS seria mais vantajoso do que uma hospedagem compartilhada.

R: Sites que superaram os limites da hospedagem compartilhada, mas ainda não exigem todo o poder de um servidor físico exclusivo.
Questão 17: Diferencie os conceitos de IP e porta em uma rede, explicando suas funções.

R: IP é um número identificador de um dispositivo na rede, já a porta é um tipo de canal de comunicação utilizado na rede. O IP identifica um dispositivo dentro da rede, já a porta permite que os dados fluam entre um cliente e um servidor pela Internet.
Questão 18: Qual a importância de um servidor web conseguir lidar com conexões via HTTPS?

R: Para garantir a integridade, segurança e confidencialidade dos dados que são enviados a tal.
Questão 19: Cite dois exemplos de servidores web e explique suas principais características.

R: LiteSpeed, focado em performance e compatível com Apache; Caddy, que é um servidor moderno com HTTPS automático e configuração simples.
Questão 20: Defina o que é uma hospedagem compartilhada e em quais casos ela costuma ser utilizada.

R: É uma hospedagem na qual vários sites no mesmo servidor físico, com baixo custo. É útil em caso de sites menores e iniciantes, que necessitam apenas de pouco poder na hospedagem.
Questão 21: Explique a função de um servidor de aplicação e a diferença em relação a servidores que apenas servem arquivos estáticos.

R: Um servidor web lida com protocolos de segurança, e fornecem apenas conteúdo estático, como imagens, por exemplo. (Nota: A resposta parece incompleta, mas foi mantida como no original).
Questão 22: Cite um servidor de aplicação usado para projetos em Python e descreva como ele funciona.

R: Django (com Gunicorn/WSGI), com framework Python com servidores como Gunicorn ou uWSGI.
Questão 23: O que é o WildFly (ex-JBoss) e para qual tecnologia ele é indicado?

R: Um servidor de aplicação com plataforma corporativa da RedHat, indicado para Java.
Questão 24: Qual a função da porta 21 em redes de computadores e para qual serviço ela é utilizada?

R: A porta 21 é comumente associada ao FTP (File Transfer Protocol). O FTP serve para transferir arquivos entre um cliente e um servidor. Geralmente é considerado um protocolo de transferência de arquivos “não seguro”.
Questão 25: Explique por que o servidor LiteSpeed é compatível com o Apache e quais as vantagens dessa compatibilidade.

R: O LiteSpeed foi projetado para ser um substituto direto ("drop-in replacement") do Apache, lendo nativamente seus arquivos de configuração, incluindo o .htaccess. A principal vantagem é a migração fácil e rápida: você pode trocar de servidor web para obter um desempenho muito superior e acesso a recursos como o LSCache, sem precisar reconfigurar seu site ou reescrever regras. Isso economiza tempo, dinheiro e evita erros de configuração.
Questão 26: O que é o Microsoft IIS e em quais situações ele costuma ser adotado?

R: Servidor web da Microsoft, integrado ao Windows Server. O IIS é adotado principalmente em ambientes que usam Windows Server para hospedar aplicações desenvolvidas com tecnologias da Microsoft, como o .NET.
Questão 27: Explique para que serve o Node.js e em quais tipos de aplicações ele é mais utilizado.

R: É um servidor de aplicação com linguagem Javascript que executa código JS no servidor, muito usado em APIs e aplicações web.
Questão 28: Descreva a função do servidor GlassFish e para qual linguagem ele é destinado.

R: Servidor completo para aplicações Java EE. Usado para linguagem Java.
Questão 29: O que são Puma e Unicorn e em quais projetos eles são normalmente utilizados?

R: Puma e Unicorn são servidores de aplicação web para a linguagem Ruby, usados principalmente para executar aplicações construídas com o framework Ruby on Rails. Puma é o padrão atual, favorecido por seu modelo multi-threaded que permite alta concorrência. Unicorn, mais antigo, cria processos separados para cada requisição, sendo valorizado pela estabilidade.
Questão 30: Qual a importância da escolha adequada entre hospedagem local, compartilhada, VPS e nuvem no planejamento de um projeto web?

R: A escolha correta alinha o desempenho, a escalabilidade e o custo da infraestrutura às necessidades atuais e futuras do projeto. Uma decisão inadequada pode limitar o crescimento, gerar custos inesperados ou comprometer a experiência do usuário, impactando diretamente o sucesso da aplicação.
